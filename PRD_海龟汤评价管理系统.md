海龟汤评价管理系统 — PRD 文档



## 1. 产品概述

### 1.1 产品定位
海龟汤评价管理系统是一套面向海龟汤（情境谜题）爱好者的轻量化内容管理与评价平台。它让用户可以分享自己创作的海龟汤、对他人作品进行多维度评价，并以雷达图方式直观展示作品得分。

### 1.2 核心价值
- **分享**：创作者可以发布自己的海龟汤（汤面 + 汤底 + 主持人手册）
- **评价**：对海龟汤从总评、文笔、逻辑、分享性、机制、反转、深度六个维度进行评分
- **可视化**：六维雷达图直观展示作品综合水平
- **社区化**：注册系统 + 权限管理，形成创作者社区

### 1.3 目标用户
| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| 普通用户（未登录） | 访客/路人 | 浏览海龟汤列表、查看详情、查看评价 |
| 注册用户 | 海龟汤爱好者 | 创建海龟汤、评价作品、管理自己的内容和评价 |
| 管理员 | 平台运营者 | 用户管理、全平台内容管控 |

### 2.2 用户故事

#### 2.2.1 访客浏览（未登录）
- 用户可以打开网站，看到海龟汤列表
- 可以点击某个海龟汤查看详情（汤面、汤底、主持人手册）
- 可以查看作品的评价列表和雷达图
- **不可以**：创建汤、评价、编辑、删除

#### 2.2.2 用户注册与创作
- 用户点击「注册」，填写昵称、账号、密码
- 系统校验账号唯一性，注册成功后自动登录
- 用户可以点击「新建海龟汤」，填写富文本内容（汤面、汤底、主持人手册）
- 提交后作品在列表中展示

#### 2.2.3 用户评价
- 登录用户可以在海龟汤详情页或评价弹窗中添加评价
- 评价包括：总评分（1-5，步长0.5）+ 六个维度的可选评分
- 评价后该作品的雷达图和均分实时更新

#### 2.2.4 权限控制
- 普通用户 → 只能编辑/删除自己的汤和评价
- 管理员 → 可以编辑/删除所有汤和所有评价
- 未登录 → 只能查看，不能创建/编辑/删除/评价

## 3. 非功能需求

### 3.1 性能要求
- 页面首屏加载 < 2s
- API 响应时间 < 500ms（本地网络下 < 100ms）
- 支持 50 人并发访问

### 3.2 安全要求
- 密码使用 bcrypt 加密存储
- Session 会话管理（httpOnly cookie）
- 后端所有增删改操作进行权限校验
- SQL 注入防护（参数化查询）

### 3.3 可用性要求
- 服务 7×24 运行（依托服务器）
- 数据持久化到 MySQL，不丢失
- 前端适配手机端（<420px 单列布局）

### 3.4 兼容性要求
- 现代浏览器（Chrome/Firefox/Safari/Edge 最近2个版本）
- 移动端 Safari + Chrome 适配

## 4. 数据模型

### 4.1 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) PK | 主键 |
| username | VARCHAR(50) UNIQUE | 登录账号 |
| password | VARCHAR(128) | bcrypt 加密密码 |
| nickname | VARCHAR(50) | 显示名称 |
| role | ENUM('admin','user') | 用户角色 |
| created_at | DATETIME | 创建时间 |

### 4.2 海龟汤表 (soups)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) PK | 主键 |
| title | VARCHAR(200) | 汤标题 |
| author | VARCHAR(100) | 作者名 |
| type | VARCHAR(20) | 类型（本格清汤/红汤/黑汤/变格...） |
| surface | TEXT | 汤面（富文本 HTML） |
| bottom | TEXT | 汤底（富文本 HTML） |
| manual | TEXT | 主持人手册（富文本 HTML，选填） |
| creator_id | VARCHAR(64) FK | 创建者 ID |
| creator_name | VARCHAR(50) | 创建者昵称（冗余） |
| created_at | DATETIME | 创建时间 |

### 4.3 评价表 (evaluations)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) PK | 主键 |
| soup_id | VARCHAR(64) FK | 关联海龟汤 |
| total | DECIMAL(3,1) | 总评分（1-5） |
| reviewer | VARCHAR(50) | 评价人名称 |
| reviewer_id | VARCHAR(64) FK | 评价人 ID |
| writing | DECIMAL(3,1) | 文笔（0-5，0表示未评） |
| logic | DECIMAL(3,1) | 逻辑 |
| share | DECIMAL(3,1) | 分享性 |
| mechanism | DECIMAL(3,1) | 机制 |
| twist | DECIMAL(3,1) | 反转 |
| depth | DECIMAL(3,1) | 深度 |
| created_at | DATETIME | 评价时间 |

## 5. 界面原型说明

### 5.1 首页 → 海龟汤列表
- 顶部导航栏：Logo + 标题 + 用户信息 + 登录/注册/退出按钮
- 操作按钮：新建海龟汤（登录后可见）
- 表格列表：雷达图缩略图 | 标题 | 作者 | 类型 | 创建者 | 评价数 | 均分 | 操作按钮
- 响应式适配手机

### 5.2 详情页
- 元信息卡片（标题/作者/类型/创建者/时间/评价数/均分）
- 汤面内容区（富文本渲染）
- 汤底内容区（富文本渲染）
- 主持人手册区（富文本渲染，有则显示）
- 六维雷达图（Chart.js 渲染）
- 评价列表 + 添加评价入口

### 5.3 模态框
- 登录/注册弹窗（切换模式）
- 新建/编辑海龟汤弹窗（富文本编辑器）
- 添加/编辑评价弹窗（维度评分滑块/输入）
- 评价列表弹窗（含该汤所有评价）
- 用户管理弹窗（管理员专用）

## 6. 验收标准

| # | 验收项 | 判定条件 |
|---|--------|---------|
| 1 | 用户注册 | 填写昵称/账号/密码后可注册，重复账号提示 |
| 2 | 用户登录/退出 | 登录后可操作，退出回到访客状态 |
| 3 | 创建海龟汤 | 登录后填写富文本内容可创建，列表可见 |
| 4 | 编辑海龟汤 | 仅创建者和管理员可编辑 |
| 5 | 删除海龟汤 | 仅创建者和管理员可删除，级联删除评价 |
| 6 | 未登录限制 | 未登录看不到新建/编辑/删除/评价按钮 |
| 7 | 添加评价 | 登录后可对任意汤添加评价 |
| 8 | 雷达图 | 有评价数据时显示雷达图，无数据提示 |
| 9 | 管理员后台 | 管理员可管理所有用户 |
| 10 | 手机适配 | 手机浏览器正常使用，布局自适应 |
| 11 | 数据持久化 | 重启服务后数据不丢失（MySQL） |
