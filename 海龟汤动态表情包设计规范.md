# 海龟汤动态表情包设计规范

> 文档版本：V1.1  
> 适用范围：海龟汤移动端 Web 私信、游戏互动、活动宣传  
> 目标数量：20个动态表情  
> 系列名：**汤汤侦探**  
> 核心角色名：**汤汤**

---

## 1. 项目目标

本项目拟打造一套具有统一角色、统一画风和明确海龟汤辨识度的动态表情包，用于提升应用内的互动氛围、用户表达效率和品牌记忆。

整套表情包应同时满足以下目标：

1. **可爱好用**：适合年轻用户，能够自然用于私信和游戏互动。
2. **海龟汤特色明确**：每个表情至少体现一个谜题、推理、汤碗、线索或破案相关元素。
3. **统一可扩展**：所有表情属于同一角色体系，后续可继续增加节日、成就、活动等系列。
4. **动态轻量化**：正式成品统一使用 Animated WebP，不采用 GIF；仅在明确需要矢量交互并通过技术评估时使用 Lottie。
5. **手机端友好**：保证透明背景、加载迅速、循环自然，在手机端小尺寸下仍清晰易读。

### 2.1 系列定位

| 项目 | 规范 |
|---|---|

| 核心关键词 | 萌系、神秘、聪明、俏皮、轻悬疑 |

| 使用氛围 | 轻松互动、烧脑推理、反转惊喜、通关庆祝 |

### 2.2 核心角色设定

#### 角色名称
**汤汤**

#### 角色身份
一只喜欢喝汤、寻找线索、破解谜题的Q版小乌龟侦探。

#### 性格标签
- 好奇心强
- 有点傲娇
- 聪明但偶尔犯迷糊
- 遇到谜题会立刻进入侦探状态
- 害怕恐怖故事，但又忍不住继续听

#### 外形特征
- 2至2.5头身的Q版比例
- 圆头、短手短脚、软萌轮廓
- 大眼睛，小嘴巴，带淡淡腮红
- 龟壳带有月牙或旋涡状谜题纹样
- 固定携带迷你放大镜或汤勺
- 表情变化明显，但不破坏基础脸型

#### 固定识别元素
以下元素至少保留三项，确保角色识别稳定：

1. 薄荷青绿色身体
2. 深夜紫龟壳
3. 棕色侦探帽子
4. 星月形龟壳纹样
5. 迷你放大镜
6. 小汤碗

## 3. 整体视觉风格

### 3.1 风格关键词

**软萌圆润、甜酷梦幻、神秘推理、低饱和高质感**

### 3.2 画面原则

- 主体轮廓圆润，禁止尖锐、硬朗或写实结构。
- 表情夸张但不失可爱。
- 每张表情只表达一个核心情绪或动作。
- 优先使用透明背景，不制作完整场景背景。
- 点缀元素数量控制在1至4个，避免视觉杂乱。
- 不使用复杂写实光影，不使用大面积渐变噪点。
- 小尺寸预览时，角色、文案和动作意图必须一眼可辨。

### 3.3 禁止风格

- 禁止血腥、尸体、武器、伤口等强刺激元素。
- 禁止写实恐怖、猎奇、压抑或令人不适的表情。
- 禁止不同表情使用不同画风或不同头身比例。
- 禁止过度使用黑金风、赛博风或厚重金属质感。
- 禁止复杂背景抢夺主体注意力。
- 禁止每张表情都更换角色服装和基础配色。
- 禁止依赖细碎纹理表达关键内容。

## 4. 色彩规范

### 4.1 主色卡

| 颜色名称 | 色值 | 使用场景 |
|---|---|---|
| 薄荷青 | `#75D8C4` | 角色身体、主要识别色 |
| 奶油白 | `#FFF7E8` | 眼白、蒸汽、文字高亮 |
| 深夜紫 | `#403453` | 龟壳、轮廓、神秘元素 |
| 蜜桃粉 | `#F5A6B8` | 腮红、爱心、可爱元素 |
| 星光黄 | `#FFD66B` | 灯泡、星星、成功反馈 |
| 雾霭蓝 | `#91BDE5` | 谜雾、冷静、思考状态 |
| 侦探棕 | `#8B6645` | 固定侦探帽、少量道具 |
| 墨紫灰 | `#2F2938` | 文字描边、深色轮廓 |

### 4.2 色彩使用比例

建议单张表情遵循：

- 主角色色：60%至75%
- 深色轮廓及龟壳：15%至25%
- 星光黄等高亮色：不超过8%
- 其他辅助色：不超过10%

### 4.3 色彩限制

- 单张表情主要颜色不超过6种。
- 高饱和色仅用于强调，不得铺满大面积。
- 不同表情中角色身体、龟壳和固定装饰色不可随意变化。
- 文字颜色应保持统一，优先使用奶油白填充和墨紫灰描边。

## 5. 线条与造型规范

### 5.1 轮廓线

- 线条圆润、平滑、无明显毛刺。
- 外轮廓线略粗，内部细节线略细。
- 512×512源文件建议外轮廓线宽为8至14px。
- 内部细节建议线宽为4至8px。
- 轮廓颜色优先使用墨紫灰，不使用纯黑色。
- 动画过程中线条粗细不得明显跳变。

### 5.2 角色比例

| 项目 | 建议比例 |
|---|---|
| 头部 | 角色总高的40%至50% |
| 身体 | 角色总高的25%至35% |
| 龟壳 | 角色宽度的40%至55% |
| 眼睛 | 脸部宽度的25%至35% |
| 手脚 | 短小圆润，避免过长 |
| 配件 | 不超过角色面积的30% |

### 5.3 表情设计

- 眼睛是核心情绪载体，应优先表现眨眼、瞪大、半眯、转圈、泪光等变化。
- 嘴型保持简单，避免复杂唇形。
- 腮红可用于害羞、开心、紧张等状态，但不应每张都保持完全相同。
- 头顶问号、感叹号、灯泡等符号应采用统一造型。

## 6. 文案规范

### 6.1 文案长度

- 推荐字数：2至5个汉字。
- 最大字数：不超过6个汉字。
- 优先使用口语化、可直接聊天发送的短句。
- 避免使用完整长句和解释性文字。

### 6.2 文案语气

文案应符合以下语气：

- 可爱但不幼稚
- 俏皮但不油腻
- 甜酷但不高冷
- 符合海龟汤玩家的互动习惯
- 避免网络热词过度依赖，防止快速过时

### 6.3 字体规范

- 使用圆体、黑体或手写感较弱的可爱字体。
- 必须确认字体允许商业使用。
- 整套表情使用同一款主字体。
- 字体不得过细，缩小后应保持清晰。
- 推荐奶油白字面、墨紫灰描边。
- 描边宽度建议为字高的6%至10%。
- 文字可以轻微弹跳、缩放或晃动，不做复杂逐字特效。

### 6.4 文字位置

- 文字原则上置于角色上方、左上方或右上方。
- 不遮挡眼睛、嘴巴、放大镜和汤碗。
- 文字与画布边缘保留至少24px安全距离。
- 同一类表情尽量保持相近的文字位置。

- 不使用真实血液、刀具、手铐等刑侦写实元素。

## 8. 动画设计规范

### 8.1 动画目标

动画应做到：

- 第一眼看懂动作意图
- 循环自然
- 体积轻量
- 小尺寸仍有表现力
- 角色不发生明显变形或结构漂移

### 8.2 推荐动画结构

每个表情采用三段式结构：

1. **预备动作**：0.1至0.3秒  
2. **核心动作**：0.6至1.2秒  
3. **缓冲与回位**：0.3至0.8秒  

### 8.3 推荐参数

| 项目 | 规范 |
|---|---|
| 动画时长 | 1.2至2.4秒 |
| 推荐帧率 | 12fps |
| 可接受帧率 | 10至15fps |
| 循环方式 | 无缝循环或完整播放后停顿再循环 |
| 循环停顿 | 0.2至0.5秒 |
| 单次主动作 | 1个 |
| 辅助动作 | 0至2个 |
| 动画曲线 | 缓入缓出、轻微回弹 |
| 角色位移 | 不超过画布宽度的15% |
| 旋转角度 | 通常不超过15° |
| 缩放幅度 | 90%至115% |

### 8.4 推荐轻量动作

- 眨眼
- 挥手
- 点头
- 歪头
- 轻跳
- 小幅后仰
- 呼吸起伏
- 耳饰或斗篷摆动
- 问号弹出
- 星星闪烁
- 灯泡亮起
- 汤碗冒蒸汽
- 放大镜闪光
- 纸条展开
- 小皇冠落下

### 8.5 动作限制

- 不同时进行超过3组独立动画。
- 不使用大面积粒子爆炸。
- 不使用整屏高频闪烁。
- 不使用快速抖动超过0.5秒。
- 不使用复杂镜头移动。
- 不做连续长距离奔跑或大角度旋转。
- 不依赖高帧率弥补动作设计问题。

### 8.6 循环原则

优先采用以下循环方式：

#### A. 无缝循环
适合：
- 眨眼
- 呼吸
- 蒸汽
- 摇头
- 害怕发抖

#### B. 动作完成后停顿循环
适合：
- 破案成功
- 灯泡亮起
- 一击命中
- 汤面奉上
- 引导互动

建议主动作完成后停留0.3秒，再回到起始状态。

## 9. 文件与输出规范

### 9.1 输出格式优先级

#### 第一优先：Animated WebP

适合当前移动端 Web 私信和应用互动场景。

优点：

- 支持透明背景
- 颜色质量优于GIF
- 体积较小
- 可实现流畅循环
- 制作与预览方便

#### 例外格式：Lottie JSON

仅适合确有主题换色、进度控制、状态切换等交互需求的矢量动画，不作为本系列常规交付物。

优点：

- 文件体积更小
- 缩放清晰
- 动画流畅
- 便于开发控制播放次数和速度

限制：

- 不适合复杂笔刷、纹理、模糊、噪点和复杂遮罩。
- Web 端需要额外播放器和渲染运行时。
- 正式采用前必须由开发确认必要性、兼容性和播放器成本。

#### 备选：APNG

仅在目标环境不支持Animated WebP时使用。

#### 禁止：GIF

不作为正式交付格式，原因包括：

- 文件体积偏大
- 透明边缘质量较差
- 色彩数量受限
- 同等体积下流畅度和画质较低

### 9.2 画布与尺寸

| 文件类型 | 尺寸 |
|---|---|
| 原始设计稿 | 512×512px |
| 标准输出 | 320×320px |
| 小尺寸输出 | 240×240px |
| 高清备用 | 512×512px |

### 9.3 安全区

- 角色及主要元素应控制在画布中央80%范围内。
- 四周至少保留24px安全距离。
- 顶部文字不得超出上方安全区。
- 汤碗蒸汽、问号等运动元素需预留完整运动空间。
- 动画任何帧均不得发生主体被裁切。

### 9.4 透明背景

- 输出文件必须为透明背景。
- 角色边缘不得出现白边、黑边或半透明脏边。
- 半透明蒸汽和光效应在深浅背景下分别测试。
- 不在透明底上使用过弱的浅色细线。

### 9.5 文件体积

#### Animated WebP

| 等级 | 单个文件建议 |
|---|---|
| 理想体积 | 150KB至300KB |
| 可接受体积 | 300KB至500KB |
| 最大上限 | 500KB |

#### Lottie JSON

| 等级 | 单个文件建议 |
|---|---|
| 理想体积 | 30KB至100KB |
| 可接受体积 | 100KB至180KB |
| 最大上限 | 200KB |

### 9.6 压缩原则

- 优先减少无意义帧和重复细节。
- 优先减少大面积透明渐变和复杂模糊。
- 优先采用角色局部运动，而非整张画面重绘。
- 动画帧率保持在10至15fps。
- 输出前删除不可见图层、未使用资源和隐藏素材。
- Animated WebP在不明显损伤边缘的前提下，可使用有损压缩。
- 压缩后必须再次检查文字边缘、眼睛和轮廓线。

## 10. 源文件规范

### 10.1 源文件类型

根据制作方式，可交付：

- Adobe Illustrator：`.ai`
- Adobe After Effects：`.aep`
- Figma源文件
- Spine工程文件
- SVG矢量源文件
- 分层PSD：`.psd`

### 10.2 分层结构

每个表情源文件至少按以下结构分层：

```text
01_角色
├── 头部
├── 眼睛
├── 眉毛
├── 嘴巴
├── 腮红
├── 身体
├── 左手
├── 右手
├── 左脚
├── 右脚
├── 龟壳
└── 固定装饰

02_道具
├── 放大镜
├── 汤碗
└── 其他道具

03_特效
├── 星星
├── 问号
├── 蒸汽
└── 光效

04_文案

05_参考与辅助线
```

### 10.3 图层命名

- 统一使用中文或英文，不混用无意义默认名称。
- 禁止出现“图层1”“组23”“副本5”等不可读命名。
- 左右部件必须标记L/R或左/右。
- 所有可动画部件单独分层。
- 旋转部件需正确设置中心点。

## 14. 应用内展示规范

### 14.1 使用位置

可用于：

- 私信输入区与消息列表
- 海龟汤游戏房间
- AI玩汤互动
- 发布成功反馈
- 通关结果页
- 成就解锁弹窗
- 用户主页表情收藏
- 活动宣传页面

### 14.2 展示尺寸

| 场景 | 建议显示尺寸 |
|---|---|
| 表情选择面板 | 72×72px至96×96px |
| 聊天气泡内 | 120×120px至180×180px |
| 通关结果页 | 180×180px至240×240px |
| 成就弹窗 | 120×120px至160×160px |
| 宣传页面 | 240×240px至320×320px |

### 14.3 播放规则

- 表情进入可视区域后再播放。
- 离开可视区域后暂停，减少性能消耗。
- 同一屏幕同时播放的动态表情建议不超过6个。
- 表情选择面板可显示静态首帧，点击或长按后预览动画。
- 聊天列表中优先播放2至3次后停留在结束帧。
- 重要反馈类表情可完整播放一次，不强制无限循环。
- 开发应为低性能设备提供静态首帧降级方案。
- Animated WebP 不承担精确暂停或逐帧控制；选择面板默认显示静态首帧，打开预览或发送后再加载动态资源。

### 14.4 加载与缓存

- 首屏只预加载最常用的6至10个表情。
- 其余表情采用懒加载。
- 动态资源应使用本地缓存或CDN缓存。
- 同一资源不得重复下载。
- 加载失败时显示透明底静态 WebP 首帧。

## 15. 设计制作流程

### 阶段一：角色定稿

交付内容：

- 正面、侧面、背面三视图
- 标准站姿
- 8种基础表情
- 固定配件
- 色卡
- 线条规范

确认后不得在单张表情中随意改变角色基础设定。

### 阶段二：静态分镜

每个表情先完成：

- 起始帧
- 核心动作帧
- 结束帧
- 文字位置
- 特效位置

静态分镜通过后再进入动画制作。

### 阶段三：动画小样

先制作以下5个代表性表情：

1. 你好呀
2. 我有问题
3. 好烧脑呀
4. 破案成功
5. 开始喝汤

这5个分别覆盖挥手、弹出、晕圈、跳跃和蒸汽等主要动画类型，可用于验证整体风格和技术方案。

### 阶段四：批量制作

- 按统一脚本制作剩余15个。
- 每完成5个进行一次风格复核。
- 不允许全部完成后才统一调整。

### 阶段五：压缩与适配

- 输出320×320 Animated WebP。
- 压缩至目标体积。
- 对浅色、深色和复杂背景进行测试。
- 在主流移动端浏览器和微信内置浏览器中验证。

交付内容：

## 16. 验收标准

### 16.1 视觉验收

- [ ] 20个表情均为同一角色和同一画风。
- [ ] 角色身体、龟壳和固定装饰颜色一致。
- [ ] 每张表情均有明确的海龟汤相关元素。

- [ ] 同屏播放时无明显卡顿。

- [ ] 正式成品不使用GIF格式。
- [ ] Animated WebP单个文件不超过500KB。
- [ ] Lottie单个文件原则上不超过200KB。
- [ ] 所有文件命名符合规范。
- [ ] 输出尺寸与约定一致。

## 17. 优化文件体积的方法

按优先级依次执行：

1. 删除不必要的空白帧。
2. 将帧率控制为12fps左右。
3. 将大面积动作改为局部动作。
4. 减少粒子数量和透明渐变。
5. 合并重复图层和重复资源。
6. 减少复杂模糊、阴影和光晕。
7. 将角色固定部分设为复用图层。
8. 缩短动画时长。
9. 降低不可见区域的画面精度。
10. 使用静态停顿代替连续重复动作。
11. 对Animated WebP进行适度有损压缩。
12. 在Lottie中删除未使用的形状和关键帧。

不应通过以下方式强行压缩：

- 将尺寸压得过小导致文字模糊。
- 将帧率降至8fps以下。
- 严重损伤角色轮廓和眼睛细节。
- 删除关键动作帧导致动画生硬。
- 使用明显色块或噪点破坏画风。
